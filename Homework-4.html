Question 1: What is Tidy Data?

Hadley Wickham’s paper defines tidy data as a standard way of organizing datasets to make analysis easier. In other words, a dataset is tidy when it follows three rules: 1) every variable forms a column; 2) every observation forms a row; 3) and every type of observational unit forms its own table. This structure makes it straightforward to filter, transform, aggregate, and visualize data because each variable is always in a consistent, predictable location.

Wickham points out that most real-world datasets are messy, and identifies five common problems: 1) column headers that are values instead of variable names; 2) multiple variables stored in one column; 3)variables stored in both rows and columns, 4) multiple types of observations in one table, 5) and one type of observation spread across multiple tables. Each of these problems requires a different tidying technique, such as melting (stacking columns into rows) or casting (rotating rows back into columns).

The practical importance of tidy data is that it works seamlessly with tidy tools functions that take tidy input and return tidy output. So, for example, in R, packages like ggplot2 and dplyr are designed around tidy data, meaning that once your data is tidy, analysis and visualization become much simpler and faster.

# Load COVID-19 data from GitHub URL
covid_df <- read.csv(
  "https://raw.githubusercontent.com/owid/covid-19-data/master/public/data/owid-covid-data.csv",
  header = TRUE
)

# Convert date column to Date type
covid_df$date <- as.Date(covid_df$date)

# Filter to Europe + population > 1 million
europe_df <- covid_df %>%
  filter(continent == "Europe") %>%
  filter(population > 1000000)

# Create month column
europe_df <- europe_df %>%
  mutate(month = floor_date(date, "month"))

# Summarise cases per month per country
plot_data <- europe_df %>%
  group_by(location, month) %>%
  summarise(cases_per_month = sum(new_cases_per_million, na.rm = TRUE)) %>%
  ungroup()
## `summarise()` has regrouped the output.
## ℹ Summaries were computed grouped by location and month.
## ℹ Output is grouped by location.
## ℹ Use `summarise(.groups = "drop_last")` to silence this message.
## ℹ Use `summarise(.by = c(location, month))` for per-operation grouping
##   (`?dplyr::dplyr_by`) instead.
# Find top 6 countries by April cases
top6_april <- plot_data %>%
  filter(format(month, "%m") == "04") %>%
  group_by(location) %>%
  summarise(april_total = sum(cases_per_month, na.rm = TRUE)) %>%
  arrange(desc(april_total)) %>%
  slice_head(n = 6)

# Add flag column for April top 6
plot_data <- plot_data %>%
  mutate(april_high = ifelse(location %in% top6_april$location, 1, 0))

# Plot: April highlight
ggplot(plot_data,
       aes(x = month, y = cases_per_month,
           group = location, color = location)) +
  geom_line() +
  gghighlight(april_high == 1, use_direct_label = FALSE) +
  scale_y_continuous(trans = "log2") +
  labs(title = "COVID-19 Cases per Month - European Countries (April Highlight)",
       x = "Month", y = "Cases per Million (log2 scale)")
